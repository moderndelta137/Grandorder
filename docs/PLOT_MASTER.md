## 0. 目的

本書は、V1シナリオ実装における**物語進行の統括管理表**である。  
以下を一元管理し、本文ライティング前後の齟齬を防ぐ。

1. 各章で「いつ / どこで / 誰が / 何に遭遇するか」
2. 分岐開始点と、分岐ごとの方向性（合理 / 信義 / 野心）
3. 固定シナリオ（章本文）と通常夜戦（システム戦闘）の接続規則
4. エンディング到達までの因果トレース

## 1. 運用原則（ライティング開始前の拘束条件）

### 1.1 シーン種別
- **固定シナリオ**: `chapter*_main_*` で管理する本文イベント。
- **通常行動**: `dayAction` / `nightBattle` などのシステムイベント。
- **橋渡しシーン**: 固定→通常、通常→固定を接続する短尺本文。

### 1.2 粒度
- 1シーンは「1主目的 + 1意思決定 + 1結果示唆」を基本単位とする。
- 画面スクロールを抑えるため、長文は複数シーンに分割する。

### 1.3 時間・場所の記法（本書の「いつ」「どこ」）
- 「いつ」は作中日時（例: `10/1 23:10`）を指す。
- 「どこ」はシーンの主舞台（例: 主人公工房 / 北区画路地 / 病院外周）を指す。
- 1章内でも、時間経過と場所遷移を明示して連続性を作る。

### 1.4 表示文と設計文の分離
- プレイヤー表示文にはメタ説明（章番号運用、実装都合、仕様注釈）を書かない。
- 実装都合や管理メモは、コードコメント・本ドキュメントに保持する。

### 1.5 呼称ルール（V1固定）
- サーヴァント → 主人公: **「マスター」**
- 主人公 → サーヴァント: **クラス名呼び（例: セイバー）**


### 1.6 「システム層」表記の運用
- 本書での「システム層」は、`dayAction` / `dayEncounterCheck` / `nightBattle` / `chapterEncounter` など、本文外で進むロジック処理を指す。
- `chapterEncounter` 等で敵陣営との接触を想定する区間は、登場人物欄に **敵サーヴァント/敵マスター（直接 or 間接）** を明記する。
- 「間接」は姿を見せない通信・使い魔・結界越し干渉を含む。


### 1.8 ARPGシナリオロジック規律（本スレッド適用）
- 本設計は「小説」ではなく **Scenario Logic Design** として扱う。
- キャラクター行動は必ず **Core Desire（達成したいこと）** と **Core Fear（避けたい破綻）** で説明可能にする。
- 戦闘は埋め草禁止。必ず物語進行（状態変化）を伴う。
- 感情独白はイベント起因時のみ最小限に許容する。

### 1.9 戦闘シーン必須チェック（全戦闘区間）
戦闘を含む区間（`chapter*_branch_*` / `nightBattle`接続）は必ず以下2点を記述する。
1. **WHY fight now?**（なぜ今戦うのか）
2. **WHAT changes after battle?**（戦闘後に何が変化するか）

### 1.7 登場人物固定と因果駆動テンプレート（全章共通）

#### 1.7A 固定登場人物ロール
- **プレイヤー側（固定）**: 主人公マスター / 自サーヴァント
- **NPC側（固定）**: 監督役（教会） / 協力者（一般魔術師・現地支援者）
- **敵側（固定）**: 敵マスター（意思決定者） / 敵サーヴァント（実行者）

#### 1.7B 因果駆動テンプレート（各シーンで必須）
各シーン（とくに `chapter*_branch_*`）は、以下を順に満たす。
1. **敵意図（Why）**: 敵がその場で何を得たいか（主導権/露見誘発/撤退線遮断など）
2. **敵行動（How）**: 誰が主導し、どの手段を使うか（対軍圧/追撃/攪乱）
3. **プレイヤー対抗（Counter）**: 主人公が選べる対応（回避/強行/令呪/共闘）
4. **代償（Cost）**: 対応の結果、失うもの（魔力/令呪/被害/同盟/拠点）
5. **持越状態（State）**: 次シーンへ持ち越す値（`idealPoints`/`civilianDamage`/`allianceState`/`finalLockState`）

#### 1.7C 章運用ルール
- 章頭で「敵側主導者（敵マスター）」を明記し、章内で一貫させる。
- 章末シーンには「その章で確定した代償」を1つ以上記録する。
- `dayAction` を挟む区間は、前後で「時間経過に伴う目的変化」を補足する。

## 2. 物語構造マップ（章横断）

### 2.1 大局フロー
1. 契約成立（序盤）
2. 初期接敵と生存方針確立（第1章）
3. 同盟/不信の政治戦（第2章）
4. 戦術侵攻と消耗管理（第3章）
5. 中盤終端・不可逆移行（第4章）
6. 同盟崩壊と露見管理（第5章）
7. 願いの選択と結末判定（第6章）

### 2.2 システムと物語の対応
- 物語が提示する対立は、必ず既存パラメータへ還元する。
  - 理想寄り: `idealPoints` 上昇寄り
  - 強行寄り: `civilianDamage` / `trueNameExposure` 上昇寄り
  - 計画寄り: `tacticalAdvantage` / `mana` 制御

## 5. 通常夜戦と固定シナリオの統合ルール

### 5.1 基本ルール
- 固定シナリオは「意味づけ」、通常夜戦は「結果確定」を担当。
- 同一テーマを重複説明しない。
  - 固定: なぜ戦うか
  - 夜戦: どう戦ったか

### 5.2 差し込みルール
- 章本文内に短い戦況差し込みを1回以上配置する。
- 夜戦結果ログは次の固定本文冒頭で1〜2行回収する。

### 5.3 破綻防止
- 固定本文で確定したことを、直後のシステムで否定しない。
- 否定が必要な場合は「敵の介入」等の理由シーンを必ず挿入。
