## 0. 目的

本書は、V1シナリオ実装における**物語進行の統括管理表**である。  
以下を一元管理し、本文ライティング前後の齟齬を防ぐ。

1. 各章で「いつ / どこで / 誰が / 何に遭遇するか」
2. 分岐開始点と、分岐ごとの方向性（合理 / 信義 / 野心）
3. 固定シナリオ（章本文）と通常夜戦（システム戦闘）の接続規則
4. エンディング到達までの因果トレース

---

## 1. 運用原則（ライティング開始前の拘束条件）

### 1.1 シーン種別
- **固定シナリオ**: `chapter*_main_*` で管理する本文イベント。
- **通常行動**: `dayAction` / `nightBattle` などのシステムイベント。
- **橋渡しシーン**: 固定→通常、通常→固定を接続する短尺本文。

### 1.2 粒度
- 1シーンは「1主目的 + 1意思決定 + 1結果示唆」を基本単位とする。
- 画面スクロールを抑えるため、長文は複数シーンに分割する。

### 1.3 時間・場所の記法（本書の「いつ」「どこ」）
- 「いつ」は作中日時（例: `10/1 23:10`）を指す。
- 「どこ」はシーンの主舞台（例: 主人公工房 / 北区画路地 / 病院外周）を指す。
- 1章内でも、時間経過と場所遷移を明示して連続性を作る。

### 1.4 表示文と設計文の分離
- プレイヤー表示文にはメタ説明（章番号運用、実装都合、仕様注釈）を書かない。
- 実装都合や管理メモは、コードコメント・本ドキュメントに保持する。

### 1.5 呼称ルール（V1固定）
- サーヴァント → 主人公: **「マスター」**
- 主人公 → サーヴァント: **クラス名呼び（例: セイバー）**

## 2. 物語構造マップ（章横断）

### 2.1 大局フロー
1. 契約成立（序盤）
2. 初期接敵と生存方針確立（第1章）
3. 同盟/不信の政治戦（第2章）
4. 戦術侵攻と消耗管理（第3章）
5. 中盤終端・不可逆移行（第4章）
6. 同盟崩壊と露見管理（第5章）
7. 願いの選択と結末判定（第6章）

### 2.2 システムと物語の対応
- 物語が提示する対立は、必ず既存パラメータへ還元する。
  - 理想寄り: `idealPoints` 上昇寄り
  - 強行寄り: `civilianDamage` / `trueNameExposure` 上昇寄り
  - 計画寄り: `tacticalAdvantage` / `mana` 制御

## 3. 章別プロット詳細（遭遇 / 分岐 / 接続）

## 第1章: 契約初動と初夜戦準備

### 3.1A 遭遇設計（いつ・どこで・なに）
- いつ: 10/1 23:00〜10/2 00:10（契約直後〜初夜戦直前）
- どこで: 主人公工房 → 北区画路地 → 病院外周監視点
- 遭遇:
  1. 契約直後の心理的不安（主人公）
  2. サーヴァントの現実的助言
  3. 敵陣営の初期観測行動（本命前の牽制）
  4. 避難導線維持と接敵準備の同時進行

### 3.1B 分岐開始点
- 分岐点1: 契約方針（信頼 / 統制 / 短期成果）
- 分岐点2: 初夜戦方針（被害抑制 / 短期決着）

### 3.1C 分岐方向
- 信義寄り: 被害抑制・理想点確保
- 合理寄り: 統制・戦術優位確保
- 野心寄り: 短期成果・露見/被害リスク増

### 3.1D 固定と通常夜戦の組み合わせ
- 固定本文: 契約余熱・戦況差し込み・方針選択
- 通常夜戦: 実際の戦闘解決（`nightBattle`）
- 接続規則: 第1章固定シーン最終選択後に `dayAction` 経由で夜戦へ


### 3.1E シーン接続設計（時刻・場所遷移）
| シーン | 作中時刻 | 主舞台 | 目的 | 次シーン接続 |
|---|---|---|---|---|
| chapter1_main_001 | 10/1 23:00 | 主人公工房・召喚室 | 契約余熱と初期不安を提示 | 方針確認へ |
| chapter1_main_002 | 10/1 23:12 | 主人公工房・作戦机 | 北区画異常の把握、移動判断 | 速報確認へ |
| chapter1_main_003 | 10/1 23:24 | 北区画へ向かう路地 | 契約方針（信頼/統制/短期）決定 | 外周監視点へ |
| chapter1_main_004 | 10/1 23:38 | 病院外周監視点 | 敵観測行動と避難導線の同時確認 | 最終判断へ |
| chapter1_main_005 | 10/1 23:52 | 病院外周・封鎖線手前 | 初夜戦方針（被害抑制/短期決着）決定 | 出撃導入へ |
| chapter1_main_006 | 10/2 00:05 | 北区画接敵導線 | 出撃確定と心理収束 | `dayAction` → 夜戦 |

## 5. 通常夜戦と固定シナリオの統合ルール

### 5.1 基本ルール
- 固定シナリオは「意味づけ」、通常夜戦は「結果確定」を担当。
- 同一テーマを重複説明しない。
  - 固定: なぜ戦うか
  - 夜戦: どう戦ったか

### 5.2 差し込みルール
- 章本文内に短い戦況差し込みを1回以上配置する。
- 夜戦結果ログは次の固定本文冒頭で1〜2行回収する。

### 5.3 破綻防止
- 固定本文で確定したことを、直後のシステムで否定しない。
- 否定が必要な場合は「敵の介入」等の理由シーンを必ず挿入。

## 6. シナリオライティング手順（本書準拠）

1. 章ごとに「遭遇3点」「分岐2点」を先に確定。
2. 各分岐をパラメータ反映値へマッピング。
3. 固定シーン本文を書く（短尺、没入優先）。
4. `count_story_chars` で文字数確認。
5. `validate_sprint3_d3` / `validate_sprint4_d3` で回帰確認。
6. 本書の章テーブルへ差分を反映。
